I initially expected small spleen size to be the main driver of failure. The
full 8-case data doesn't support that: case 2 (8,588 voxels, similar size to
the worst cases) scored the *best* Dice of the set (0.655), and case 7 (the
single smallest spleen, 5,538 voxels) scored a middling 0.486. Size alone
doesn't explain the failures -- so the explanation below is based on actually
looking at the images, not on that initial hypothesis.

## What's actually going wrong

Looking at `failure_case_1.png`, `failure_case_4.png`, and `failure_case_3.png`,
the same pattern shows up in all three: the model roughly locates the spleen
(the primary predicted blob overlaps the true crescent shape in the correct
quadrant of the abdomen) but then **also predicts spurious false-positive
blobs elsewhere in the slice** -- on adjacent soft tissue near the kidney, and
in one case (case 1) a small stray detection in the image corner. Those extra
false positives are large relative to the true spleen volume, so they drag
Dice down hard even where the primary detection is reasonable.

Case 4 additionally shows a shape problem independent of the false positives:
the true spleen is an elongated crescent, but the model's primary prediction
is a rounder, more blob-like region that doesn't follow the actual organ
contour.

**Likely cause**: this checkpoint is a small UNet (channels 16/32/64/128),
trained CPU-only for 50 epochs on 64^3 patches, with no dropout and modest
augmentation. A 64^3 patch gives the network limited global context to learn
*where* organs typically sit relative to each other -- it's plausible the
network is keying on generic soft-tissue intensity/texture that the spleen
shares with adjacent structures (like the kidney) rather than learning more
position-specific or boundary-specific features. That's a hypothesis
consistent with what's visible in these images, not a proven mechanism.

**What would actually test this hypothesis**: the Tier 1 GPU retrain
(`train_spleen_colab.py`, larger 96^3 patches, deeper 5-level UNet, stronger
augmentation, 300 epochs) gives more spatial context and capacity. If the
false-positive-blob pattern shrinks in the retrained model's failure cases,
that would support the hypothesis above; if it persists, the cause is
probably something else (e.g., insufficient training data, label noise).
